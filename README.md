# Hybrid RAG Knowledge Platform for Scientific & Engineering Research

Интеллектуальная система поиска, анализа и синтеза научно-технических знаний для корпоративных R&D подразделений.

Проект разработан для построения единой базы знаний по инженерной документации, научным публикациям, экспериментальным данным и внутренним техническим отчетам. Основной целью системы является сокращение времени поиска информации, предотвращение дублирования исследований и обеспечение воспроизводимости принимаемых инженерных решений.

---

# Основные возможности

* Семантический поиск по научным и техническим документам
* Гибридный Retrieval (Embeddings + Lexical Search)
* Автоматическое извлечение сущностей и отношений
* Построение графа знаний предметной области
* Поддержка сложных многопараметрических запросов
* Генерация структурированных ответов исключительно на основании локальной базы знаний
* Автоматическое определение источников и уровня подтверждения информации
* Выявление пробелов знаний и противоречивых исследований

---

# Архитектура решения

В отличие от классических RAG-систем, данное решение использует многоуровневый конвейер обработки знаний.

```text
                   Documents
                       │
                       ▼
              Data Normalization
                       │
                       ▼
                NLP Processing
                       │
      ┌────────────────┴──────────────┐
      ▼                               ▼
Chunk Generation               Entity Extraction
      │                               │
      ▼                               ▼
 Embedding Index              Knowledge Graph
      │                               │
      └──────────────┬────────────────┘
                     ▼
            Hybrid Retrieval Engine
                     │
                     ▼
             Gap & Coverage Analysis
                     │
                     ▼
               LLM Answer Synthesis
                     │
                     ▼
      Structured Answer with References
```

# Почему это не классический RAG

Большинство современных RAG-систем используют только векторный поиск по embeddings и передают найденные документы языковой модели.

В данном проекте применяется **Hybrid Retrieval Architecture**, которая объединяет несколько независимых механизмов:

* адаптацию пользовательского запроса;
* извлечение предметных сущностей;
* гибридный поиск (семантический + лексический);
* граф знаний;
* анализ покрытия знаний;
* анализ отсутствующих комбинаций;
* генерацию ответа только по найденному контексту.

Таким образом языковая модель не используется как источник знаний и не отвечает "из памяти". Все ответы строятся исключительно на основании локальной базы документов.

# Основные компоненты

## NLP Pipeline

При загрузке документов автоматически извлекаются:

* материалы;
* технологические процессы;
* оборудование;
* параметры экспериментов;
* концентрации;
* температуры;
* скорости;
* авторы;
* организации;
* география;
* временные диапазоны.

Дополнительно выполняется нормализация терминологии и сопоставление синонимов.

---

## Hybrid Retrieval

Поиск основан на комбинации двух механизмов:

* Dense Retrieval (Embeddings)
* Lexical Retrieval (Keyword Matching)

Это позволяет одновременно учитывать смысл текста и точное совпадение специализированных технических терминов.

---

## Knowledge Graph

Параллельно индексированию документов строится граф знаний.

Он содержит следующие типы сущностей:

* Material
* Process
* Equipment
* Publication
* Experiment
* Expert
* Organization
* Facility

Граф используется для:

* анализа взаимосвязей;
* поиска связанных исследований;
* оценки покрытия знаний;
* обнаружения пробелов;
* поиска противоречий между исследованиями.

---

## LLM Synthesis

После завершения поиска языковая модель получает только найденные фрагменты документов.

На основании этого контекста формируется итоговый ответ с указанием источников.

Это значительно снижает вероятность появления галлюцинаций и обеспечивает полную воспроизводимость результатов.

# Поддерживаемые запросы

Система поддерживает запросы естественным языком, содержащие одновременно несколько условий.

Например:

> Какие способы электроэкстракции никеля применяются при скорости циркуляции католита 0.8–1.2 м/с в зарубежной практике после 2020 года?

или

> Найти исследования по кучному выщелачиванию медных руд в условиях холодного климата.

# Запуск проекта

Создать виртуальное окружение:

```bash
python -m venv .venv
```

Активировать окружение:

Linux / macOS

```bash
source .venv/bin/activate
```

Windows

```powershell
.venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить сервер:

```bash
uvicorn main:app --reload
```

# Оценка качества

Качество поиска оценивается с использованием стандартных IR-метрик:

* Recall@K
* Precision@K
* Mean Reciprocal Rank (MRR)
* nDCG
* Hit Rate

Качество генерации ответа оценивается по следующим показателям:

* Faithfulness
* Citation Accuracy
* Context Precision
* Context Recall
* Answer Relevancy

Для проверки эффективности архитектуры проводится сравнение с несколькими базовыми подходами:

* BM25
* Dense Retrieval
* Classical RAG
* Hybrid Retrieval
* Hybrid Retrieval + Knowledge Graph

---

# Преимущества решения

* Поиск осуществляется исключительно по локальной корпоративной базе знаний.
* Все ответы имеют подтвержденные источники.
* Отсутствует зависимость от внешних поисковых систем.
* Минимизируется вероятность галлюцинаций языковой модели.
* Поддерживаются сложные инженерные запросы с множеством параметров.
* Архитектура легко масштабируется на новые предметные области без изменения основных алгоритмов.
